Detailed Description

Result type.

Result is a type that represents either success (ok) or failure (err).

Using user-defined types

// my_result.h
#define X(RET, NAME, ARGS, DEF) RET NAME(ARGS);
DECLARE_RESULT(user_ok_type, user_err_type);
#undef X
// my_result.c
#define X(RET, NAME, ARGS, DEF) RET NAME(ARGS) DEF
IMPL_RESULT(user_ok_type, user_err_type);
#undef X
DECLARE_RESULT
#define DECLARE_RESULT(T, ERR)

Attention
user_ok_type and user_err_type must not contain whitespaces and special characters (const char * or int[] for example)

Macros thats uses compilers extensions

Error propagarion feature (TRY macro) use statement in expression extension. Thats cause some warnings of -padantic compilation flag. Define RESULT_PEDANTIC_SAFE and OPTION_PEDANTIC_SAFE before result.h and option.h including for toggle of error propagation feature of option and result types;
